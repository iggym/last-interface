Field notes — agent memory, 2026

Your AI agent will remember you.
The question nobody's marketing
slide asks: remember you correctly?

Every lab is racing to ship agents that feel like a colleague who's worked with you for years. The demo is always the same: it remembers your coffee order. Nobody demos what happens six months later, when it still confidently thinks you drink oat milk.

01 — the pitch vs. the file cabinet

Every agent is a colleague with amnesia

57% of organizations now have AI agents running in production, and the gap between "impressive demo" and "actually useful over months" almost always comes down to one thing: what the agent carries between sessions. · Gartner 2026 survey

It shows up in ugly, specific ways. One benchmark found agents resolving only 35% of multi-turn customer support tasks end-to-end — the single biggest failure mode being plain context loss, the agent forgetting what the customer already told it two messages ago. · Salesforce benchmark, 2025

So the industry's answer has been: give it memory. Vector databases, graph stores, summarization pipelines. Mem0, Zep, Cognee, Letta — a whole new product category built in about eighteen months, one of them growing pipeline volume 500x to over a million runs in a single year. · Cognee, 2026

Which fixes the amnesia. And creates a second, quieter problem almost nobody budgets for.

Notice the confidence bar didn't drop as fast as the stale count rose. That gap is the actual danger: an agent doesn't get quieter as it fills with bad data — it keeps answering just as fast, just as sure of itself. One engineer building on this put it bluntly: a confidently-recalled outdated memory is worse than the agent simply forgetting. · dev.to, 2026

Named model: "Confidently Wrong"

Not "the agent forgot." The opposite failure: the agent remembers, states it plainly, and is out of date. A human colleague who's unsure will hedge — "I think it was Tuesday?" A memory-backed agent retrieves a stale fact with the same tone of voice as a fresh one. The danger isn't forgetting. It's forgetting to flag what it forgot to update.

For a project that only lives three days (a one-off script, a prototype), what's the right memory architecture?

Named model: "The Filing System" (not the file)

The best-performing setups treat memory as a filing system that actively supersedes and discards, not one growing file. One builder's rule of thumb: accumulating context is easy — expiring it is the hard part nobody does. A memory store nobody prunes isn't an asset. It's a landfill with better UX. · Brennan Moore, 2026

05 — the analog test

Run the executive assistant test

Here's a framework worth stealing directly: before judging whether an agent's memory feature is helpful or invasive, imagine a human doing the exact same thing. Walk through one, choice by choice.

Named model: "The Assistant Test"

You've worked with the same human executive assistant for years. They know you take the aisle seat. That's convenient, unremarkable, earned trust. The test is finding the exact point on the same spectrum where it stops being that, and starts being something you never actually agreed to.

07 — for the builders

If you're actually shipping this

One real number, since most of this discourse is vibes: on LongMemEval's temporal reasoning tests — the ones that check whether an agent notices a fact changed, not just that it exists — Zep's temporal knowledge graph scored 63.8%, against 49.0% for mem0's more standard approach, largely because Zep explicitly tracks when facts were true. · LongMemEval benchmark, 2026

ApproachBest atWeak at
Vector storeSimple semantic recall, fast to shipMulti-hop reasoning, doesn't know a fact went stale
Graph / temporalTracking how facts change over timeMore setup, more to operate
SQL / relationalStructured facts, auditable, boring in a good wayPoor at fuzzy semantic search

The practical rule that keeps surfacing across teams doing this in production: use hybrid storage — vector for semantic search, graph or temporal for relationships and change, SQL for the facts that need to be audited. Skip all of it for anything that lives less than a few days. · byteiota, 2026
